Module 7 · nD 组合 + 工程

把它们拼起来：nD 并行与选型

真实的大模型训练从来不是单一并行，而是 DP × FSDP × TP × PP × CP × EP 的组合。一个多维 DeviceMesh 描述 GPU 布局，一套放置原则决定谁放最内、谁放最外。这是全手册的收束。

🗺️ DeviceMesh / parallel_state 🎯 TP 最内 · PP 最外 🌳 选型决策树 ⚙️ AC / fp8 / overlap

01真实训练 = 组合并行

前面六章每一种并行都解决一个特定问题，但单独用都不够：DP 不省显存、TP 不能跨机、PP 有气泡……所以实际训练把它们叠起来用，叫 3D / 4D / 5D 并行。比如一个经典配置：

  • 机内 8 卡做 TP（吃 NVLink 带宽）
  • 若干机做 PP（跨节点，通信稀疏）
  • 剩下的卡做 FSDP/DP（最外层，梯度可重叠）
  • 长序列再叠 CP，MoE 再叠 EP

关键问题变成：这成百上千张卡，谁和谁组成一个 TP 组、谁和谁组成 PP 组？

02DeviceMesh：用多维网格描述 GPU 布局

两个框架都用一个「多维网格」给每张卡分配一组坐标 $(\text{tp\_rank}, \text{dp\_rank}, \text{pp\_rank}, ...)$。以 16 卡、$\text{tp}=2,\text{dp}=4,\text{pp}=2$ 为例，全局 rank 由各维坐标和 stride 算出：

$$ \text{global\_rank} = \text{tp\_rank} + \text{dp\_rank}\times 2 + \text{pp\_rank}\times 8 $$
nD 布局的核心：每张卡一组坐标，stride 小的维（TP）相邻、走 NVLink；stride 大的维（PP）跨节点。

03放置原则：谁放最内、谁放最外

维度放哪层为什么
TP / EP最内层（NVLink）每层都通信、在关键路径、不可重叠 → 必须高带宽
CP中间层注意力的 KV 通信较密，紧贴 TP（Megatron）或折进 FSDP（torchtitan）
DP / FSDP外层每 step 一次、可与反向重叠 → 容忍较低带宽
PP最外层（跨节点）只在 stage 边界传一次激活，通信量最小

Megatron 把这套顺序固化成一个字符串 order='tp-cp-ep-dp-pp'（最左 = stride 最小 = 最内层）：

Megatron-LM · core/parallel_state.py:561（initialize_model_parallel 默认 order）
def initialize_model_parallel(..., order="tp-cp-ep-dp-pp"):
    data_parallel_size = world_size // (tp * pp * cp)   # dp 由 world_size 反推
    decoder_rank_generator = RankGenerator(
        tp=tensor_model_parallel_size, ep=1, dp=data_parallel_size,
        pp=pipeline_model_parallel_size, cp=context_parallel_size, order=order, ...)

torchtitan 用 ParallelDims 把 1 维 world mesh「拆」成多个命名子网格，tuple 顺序即放置顺序（左=最外 PP，右=最内 TP）：

torchtitan · distributed/parallel_dims.py:193
full_dense_mesh = unflatten_mesh(
    self._world_mesh,
    ("pp", "dp_replicate", "fsdp", "tp"),          # 左→右 = 外→内
    (self.pp, self.dp_replicate, fsdp, self.tp))
# fsdp = dp_shard * cp（CP 折进 FSDP 维）

顶层 parallelize_* 按固定次序逐个施加并行，FSDP 永远最后、包在最外：

torchtitan · models/llama3/parallelize.py:41（施加顺序）
if parallel_dims.cp_enabled:  apply_cp_to_forward(...)        # CP
if parallel_dims.tp_enabled:  model.parallelize(...)          # TP
if ac_config.mode != "none":  apply_ac(...)                   # 激活检查点
if model_compile_enabled:     apply_compile(...)              # torch.compile
apply_fsdp(model, dp_mesh, ...)                               # FSDP/HSDP（最外）

04选型决策树

实践选型：先用最省事的 FSDP/DP，按瓶颈逐维叠加。

05通用工程调优

  • 激活重计算（activation checkpointing）：前向不存中间激活，反向时重算——用算力换显存。可选择性只重算最贵的几层。
  • 混合精度 bf16 / fp8：计算用低精度、归约用 fp32（见 M2 的 MixedPrecisionPolicy）；fp8 进一步压算力与带宽。
  • 通信-计算重叠：贯穿全书的命脉——DP 反向边算边 all-reduce、FSDP 预取参数、PP 异步 p2p、EP 用 shared expert 盖住 all-to-all。
  • profiling：用 torch profiler / nsys 定位是通信瓶颈、气泡还是 OOM，再针对性调维度配比。

06全手册收束：一张账本，一组原语
