Module 3 · 张量并行 + 序列并行

张量并行 TP：把一层切到多卡

当单一层就大到放不下时，DP/FSDP 都救不了——得把这一层的矩阵乘法本身切开。TP 用「列切→行切」一对共轭操作，让每张卡只算一片，每层只付一次 all-reduce。序列并行 SP 再顺手把激活也按序列维切薄。

🧩 列切 → 行切，一次 all-reduce 🔀 f / g 共轭算子 📏 SP 切激活 🔌 通信密集 · TP≤8

01动机：FSDP 也救不了的「单层太大」

FSDP 把参数切到各卡，但它有个隐含前提：前向用到某层时，要把那层的完整参数 all-gather 到一张卡上来算。如果单独一层就大到一张卡放不下（比如超大 MLP、超大词表），或者这一层产生的中间激活放不下，FSDP 就无能为力了——它切的是「不同卡存不同层」，没切「一层内部的计算」。

张量并行（Tensor Parallel, TP）正是来切「一层内部」的：把一个大矩阵乘法 $Y = XW$ 沿某个维度切成几块，每张卡只算一块，最后拼起来。关键是怎么切才能让通信最少——Megatron 给出的经典答案是「列切配行切」。

02Megatron 式 TP：列切 → 行切，一次 all-reduce

以 Transformer 的 MLP（$Y = \text{GeLU}(XW_1)W_2$）为例。TP 把它拆成两个互补的切法：

  • 列切（Column Parallel）第一个矩阵 $W_1 = [W_1^{(1)}, W_1^{(2)}]$：每卡算出激活的一部分列 $\text{GeLU}(XW_1^{(i)})$，无需通信（GeLU 逐元素，可在分片上直接做）。
  • 行切（Row Parallel）第二个矩阵 $W_2 = \begin{bmatrix}W_2^{(1)}\\W_2^{(2)}\end{bmatrix}$：每卡用自己那片激活算出部分和，最后一次 all-reduce 求和得到完整输出。

这样整条 MLP 前向只在最后付一次 all-reduce，中间的激活始终是切分的，省通信也省激活显存。

Megatron 的「列切 → 行切」黄金组合：前向一次 all-reduce，反向一次 all-reduce。

f / g 共轭算子：通信藏在 autograd 里

Megatron 把这两次通信封装成一对共轭算子，是 TP 的数学核心。f 是列切层的入口，g 是行切层的出口，二者前向/反向行为互为镜像：

Megatron-LM · tensor_parallel/mappings.py:197 / :217
class _CopyToModelParallelRegion(torch.autograd.Function):   # f 算子（列切入口）
    @staticmethod
    def forward(ctx, input_, group):  return input_                       # 前向：直通
    @staticmethod
    def backward(ctx, grad_output):   return _reduce(grad_output, ...), None   # 反向：all-reduce

class _ReduceFromModelParallelRegion(torch.autograd.Function):   # g 算子（行切出口）
    @staticmethod
    def forward(ctx, input_, group):  return _reduce(input_, group)       # 前向：all-reduce
    @staticmethod
    def backward(ctx, grad_output):   return grad_output, None            # 反向：直通

🔑 共轭关系 f：前向 copy / 反向 all-reduce；g：前向 all-reduce / 反向 copy。一前一后正好各管一个方向的归约。Megatron 每个算子的 docstring 就一句话点明，例如 "forward: all reduce, backward copy"。两个具体的线性层把它们用起来：

Megatron-LM · tensor_parallel/layers.py（ColumnParallelLinear :985 / RowParallelLinear :1298）
# 列切：输入过 f 算子，输出默认保持 Shard(-1) 直接喂给下游行切层
input_parallel = copy_to_tensor_model_parallel_region(input_, group=self.tp_group)   # f
output = output_parallel   # gather_output=False，不 all-gather

# 行切：输入已是分片不再 scatter，输出过 g 算子做 all-reduce（SP 下换成 reduce-scatter）
if self.sequence_parallel:
    output_ = reduce_scatter_to_sequence_parallel_region(output_parallel, ...)  # SP
else:
    output_ = reduce_from_tensor_model_parallel_region(output_parallel, ...)    # 普通 TP: g / all-reduce

03注意力：按 head 切

注意力天然适合切——多个 head 彼此独立。把 QKV 投影列切（按 head 分组），每卡算自己负责的那几个 head；输出投影 $W_O$ 行切，最后一次 all-reduce 合并。和 MLP 完全同构：列切产生分片、行切归约。约束是 n_heads % tp == 0、n_kv_heads % tp == 0。

04词表并行：embedding 与 lm_head

词表维 $V$ 极大，embedding 和最后的 lm_head 也按词表切。embedding 时每卡只持有一段词表行，对不在本段的 token 先 mask 置零，再 all-reduce 拼出完整结果：

Megatron-LM · tensor_parallel/layers.py:283（VocabParallelEmbedding）
input_mask = (input_ < self.vocab_start_index) | (input_ >= self.vocab_end_index)
masked_input = input_.clone() - self.vocab_start_index
masked_input[input_mask] = 0                 # 不在本 rank 词表段的 token 占位 0
output_parallel = F.embedding(masked_input, self.weight)
output_parallel[input_mask, :] = 0.0         # 占位行清零
output = reduce_from_tensor_model_parallel_region(output_parallel, ...)   # all-reduce 拼完整

输出端的 lm_head 同样词表切。而它后面的交叉熵如果朴素做，需要把巨大的 $[B,S,V]$ logits 全 all-gather——这正是词表并行交叉熵 / Loss Parallel 要解决的问题。这是 TP 里最精妙的一环，本手册有独立深入篇：

05序列并行 SP：把激活也切薄

纯 TP 下，LayerNorm、Dropout 这些区域的激活在每张卡上仍是完整的（它们逐 token 独立，没被 TP 切到）。序列并行（Sequence Parallel, SP）顺手把这些区域的激活按序列维 $S$ 切分，让每卡只存 $1/\text{TP}$ 的激活——激活显存随之降到 $1/\text{TP}$。

代价是把 TP 原本的一次 all-reduce，拆成 SP 的 all-gather（进入 attention/matmul 前还原完整序列）+ reduce-scatter（出来后切回序列分片）。这恰好又是 M0 的恒等式：通信总量不变，但激活省了：

Megatron-LM · tensor_parallel/mappings.py（SP 沿 dim0=序列维）:296 / :351
# 列切层入口（SP）：沿序列维 all-gather 还原完整序列，反向 reduce-scatter
gather_from_sequence_parallel_region(...)        # forward all-gather / backward reduce-scatter
# 行切层出口（SP）：reduce-scatter 切回序列分片，反向 all-gather
reduce_scatter_to_sequence_parallel_region(...)  # forward reduce-scatter / backward all-gather

💡 SP 与 TP 是搭档 SP 不切注意力的计算（那是 TP/CP 的事），它只把 TP 已有的 all-reduce 改写成 all-gather + reduce-scatter，顺带把 LN/Dropout 区的激活切薄。所以 SP 必须和 TP 一起开，TP=1 时自动关闭。

06torchtitan：用 placement 声明，而非手写通信

torchtitan 不手写 f/g 算子，而是用 DTensor 的 placement 声明每个权重/激活怎么切，由框架推导出该插哪个集合通信。列切就是「权重 Shard(0)、输出 Shard(-1)」，行切就是「权重 Shard(1)、输出 Replicate（all-reduce）或 Shard(1)（SP 下 reduce-scatter）」：

torchtitan · models/common/decoder_sharding.py:57 / :68
def colwise_config() -> ShardingConfig:        # 列切
    return ShardingConfig(
        state_shardings={"weight": dense_param_placement(tp=Shard(0))},
        out_dst_shardings=dense_activation_placement(tp=Shard(-1)))

def rowwise_config(*, output_sp: bool = False) -> ShardingConfig:   # 行切
    out_tp = Shard(1) if output_sp else Replicate()   # SP→reduce-scatter；否则 all-reduce
    return ShardingConfig(
        state_shardings={"weight": dense_param_placement(tp=Shard(1))},
        out_dst_shardings=dense_activation_placement(tp=out_tp))

MLP 就是 w1/w3 列切、w2 行切；出口的 output_sp 一个开关切换 all-reduce / reduce-scatter——和 Megatron 的两个不同函数一一对应：

torchtitan · models/common/decoder_sharding.py:200（set_dense_ffn_sharding）
feed_forward_cfg.w1.sharding_config = colwise_config()                 # 列切
feed_forward_cfg.w3.sharding_config = colwise_config()                 # 列切
feed_forward_cfg.w2.sharding_config = rowwise_config(output_sp=enable_sp)   # 行切 → 出口一次通信

更经典、字面化的 ParallelStyle 写法在 qwen3_vl 里能看到（llama3 已改为上面的 config 驱动）：

    "attention.wq": ColwiseParallel(), "attention.wk": ColwiseParallel(), "attention.wv": ColwiseParallel(),

07为什么 TP 通信密集、一般不跨机

TP 和 DP/FSDP 有个本质区别：它的通信在前向/反向的关键路径上，无法与计算重叠。行切层必须等 all-reduce 完成才能往下走；而且每一层的 attention 出口和 MLP 出口各有一次 all-reduce（SP 下是 all-gather + reduce-scatter），一个 step 里几十上百层、数百次集合通信，通信量正比于 $\text{batch}\times\text{seq}\times\text{hidden}$。

⚠️ 放置原则 因此 TP 极度依赖带宽，通常只在单机内 NVLink/NVSwitch 域使用（典型 TP≤8，正好一台机器）。跨机做 TP 会被 InfiniBand/以太网的低带宽拖垮。超过单机的并行规模，交给通信稀疏或可重叠的 DP/FSDP/PP/CP 去承担——这条原则在 M7 nD 组合 会再次出现。

08对比与启用

Megatron-LM 显式 f/g 算子

  • ColumnParallelLinear / RowParallelLinear
  • 共轭 autograd 算子手写 all-reduce
  • SP：--sequence-parallel（需 TP>1）
  • 序列优先布局 [s, b, h]

torchtitan DTensor placement

  • ColwiseParallel / RowwiseParallel / SequenceParallel
  • Shard 放置即通信语义
  • tensor_parallel_degree + enable_sequence_parallel
  • 与 loss parallel、FSDP/PP 经 DTensor 组合
